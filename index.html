The Four Pillars of Python OOP

Encapsulation

Protect your object's state and control access through well-defined interfaces. Prevent invalid states and enable safe refactoring.

  • @property decorators for computed attributes
  • Name mangling with __private
  • Dataclasses with slots for memory efficiency
  • Validation in setters

Abstraction

Hide complex implementation details and expose only what's necessary. Create intuitive APIs that reduce cognitive load.

  • ABCs for formal interfaces
  • Protocols for structural typing
  • Clean separation of concerns
  • Dependency injection patterns

Inheritance

Create hierarchical relationships and promote code reuse. Understand when to use inheritance vs composition.

See Modern Python OOP in Action

Here's a taste of 2026-style Python OOP with type hints and modern patterns:

from dataclasses import dataclass from typing import Protocol, runtime_checkable from datetime import datetime # Protocol for abstraction @runtime_checkable class Savable(Protocol): def save(self) -> str: ... # Modern dataclass with slots @dataclass(slots=True, frozen=True) class UserProfile: username: str email: str created_at: datetime = datetime.now() # Property for validation/computation @property def email_domain(self) -> str: return self.email.split('@')[-1] # Composition over inheritance class UserRepository: def __init__(self, storage: Savable): self._storage = storage # Dependency injection def save_user(self, user: UserProfile) -> str: # Polymorphism in action return self._storage.save(user)

This example shows encapsulation (private storage), abstraction (Savable protocol), and polymorphism (any Savable works).
