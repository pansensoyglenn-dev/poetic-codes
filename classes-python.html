ðŸ“š Introduction to Python Classes

Object-Oriented Programming (OOP) is a programming paradigm that organizes code around objects rather than functions and logic. In Python, classes are the blueprint for creating objects. They encapsulate data for the object and methods to manipulate that data.

Think of a class as a cookie cutter and objects as the cookies. The class defines the shape (properties and behaviors), while objects are the individual instances created from that mold.

Why Use Classes?

  • Modularity: Break complex problems into smaller, manageable pieces
  • Reusability: Write once, use many times
  • Maintainability: Easier to update and debug
  • Abstraction: Hide complex implementation details

ðŸ”¨ Creating Your First Class

class Dog:
    """A simple Dog class"""
    
    def __init__(self, name, age):
        """Initialize dog attributes"""
        self.name = name  # Instance variable
        self.age = age    # Instance variable
    
    def bark(self):
        """Make the dog bark"""
        return f"{self.name} says: Woof!"
    
    def get_human_age(self):
        """Convert dog age to human years"""
        return self.age * 7

# Creating instances (objects)
dog1 = Dog("Buddy", 3)
dog2 = Dog("Max", 5)

# Using object methods
print(dog1.bark())           # Output: Buddy says: Woof!
print(dog2.get_human_age())  # Output: 35

Understanding the Code

  • __init__: Special method called when creating a new instance
  • self: Reference to the current instance (convention, not keyword)
  • self.name: Instance variables (unique to each object)
  • Methods: Functions defined inside a class that operate on instances

Practice Exercise 1: Create a Bank Account Class

Create a BankAccount class with:

  • Attributes: account_number, account_holder, balance
  • Methods: deposit(amount), withdraw(amount), display_balance()
  • Add validation: Cannot withdraw more than balance
class BankAccount:
    def __init__(self, account_number, account_holder, balance=0):
        self.account_number = account_number
        self.account_holder = account_holder
        self.balance = balance
    
    def deposit(self, amount):
        if amount > 0:
            self.balance += amount
            return f"Deposited ${amount}. New balance: ${self.balance}"
        return "Deposit amount must be positive"
    
    def withdraw(self, amount):
        if amount > self.balance:
            return "Insufficient funds"
        if amount > 0:
            self.balance -= amount
            return f"Withdrew ${amount}. New balance: ${self.balance}"
        return "Withdrawal amount must be positive"
    
    def display_balance(self):
        return f"Account {self.account_number} - Balance: ${self.balance}"

# Usage example
account = BankAccount("123456", "John Doe", 1000)
print(account.deposit(500))      # Deposited $500. New balance: $1500
print(account.withdraw(200))     # Withdrew $200. New balance: $1300

ðŸ§¬ Inheritance: Building on Existing Classes

Inheritance allows a class (child) to inherit attributes and methods from another class (parent). This promotes code reuse and establishes hierarchical relationships.

Types of Inheritance

  • Single Inheritance
  • Multiple Inheritance
  • Multilevel Inheritance
  • Hierarchical Inheritance
  • Hybrid Inheritance
class Animal:
    """Base class for all animals"""
    
    def __init__(self, name, species):
        self.name = name
        self.species = species
        self.alive = True
    
    def speak(self):
        return f"{self.name} makes a sound"
    
    def die(self):
        self.alive = False
        return f"{self.name} has died"


class Dog(Animal):  # Dog inherits from Animal
    """Dog class - inherits from Animal"""
    
    def __init__(self, name, breed):
        # Call parent class constructor
        super().__init__(name, species="Canine")
        self.breed = breed
        self.tricks = []
    
    def speak(self):  # Method overriding
        return f"{self.name} says: Woof!"
    
    def learn_trick(self, trick):
        self.tricks.append(trick)
        return f"{self.name} learned {trick}!"


class Cat(Animal):
    """Cat class - inherits from Animal"""
    
    def __init__(self, name, color):
        super().__init__(name, species="Feline")
        self.color = color
        self.lives = 9
    
    def speak(self):
        return f"{self.name} says: Meow!"
    
    def lose_life(self):
        if self.lives > 0:
            self.lives -= 1
        return self.lives

# Usage
buddy = Dog("Buddy", "Golden Retriever")
whiskers = Cat("Whiskers", "Tabby")

print(buddy.speak())           # Buddy says: Woof!
print(whiskers.speak())        # Whiskers says: Meow!
print(f"{buddy.name} is a {buddy.species}")  # Buddy is a Canine

Key Inheritance Concepts

  • super(): Calls parent class methods
  • Method Overriding: Child class provides specific implementation
  • Extensibility: Add new features without modifying parent
  • Polymorphism: Same method name, different implementations

ðŸ”’ Encapsulation: Data Protection

Encapsulation is the practice of hiding internal details and restricting direct access to an object's data. In Python, we use naming conventions and properties to achieve encapsulation.

class BankAccount:
    """Demonstrating encapsulation with private attributes"""
    
    def __init__(self, account_holder, initial_balance):
        self.account_holder = account_holder  # Public attribute
        self.__balance = initial_balance      # Private attribute (double underscore)
        self._transaction_count = 0         # Protected attribute (single underscore)
    
    # Getter method
    def get_balance(self):
        return self.__balance
    
    # Setter method with validation
    def set_balance(self, amount):
        if amount >= 0:
            self.__balance = amount
            self._transaction_count += 1
        else:
            raise ValueError("Balance cannot be negative")
    
    # Using property decorators (Pythonic way)
    @property
    def balance(self):
        return self.__balance
    
    @balance.setter
    def balance(self, amount):
        if amount >= 0:
            self.__balance = amount
            self._transaction_count += 1
        else:
            raise ValueError("Balance cannot be negative")
    
    @property
    def transaction_count(self):
        return self._transaction_count

# Usage
account = BankAccount("Alice", 1000)

# Accessing through getter/setter
print(account.get_balance())           # 1000
account.set_balance(1500)

# Using properties (preferred)
print(account.balance)                 # 1500
account.balance = 2000                 # Calls setter

# Trying to access private attribute directly (will cause error)
# print(account.__balance)  # AttributeError

# Name mangling - can still access but shouldn't
print(account._BankAccount__balance)  # 2000 (but don't do this!)

Access Modifiers in Python

  • Public: No underscore prefix (attribute) - accessible anywhere
  • Protected: Single underscore (_attribute) - convention for "internal use"
  • Private: Double underscore (__attribute) - name mangling prevents accidental access
  • Property Decorators: Pythonic way to create getters/setters

ðŸŽ­ Polymorphism: Many Forms

Polymorphism allows objects of different classes to be treated as objects of a common superclass. It enables flexibility and dynamic behavior.

Types of Polymorphism

  • Method Overriding: Same method name, different implementations in child classes
  • Method Overloading: Same method name, different parameters (not directly supported in Python)
  • Operator Overloading: Redefining operators for custom classes
  • Duck Typing: "If it looks like a duck and quacks like a duck, it's a duck"
class Shape:
    """Base class for all shapes"""
    
    def area(self):
        raise NotImplementedError("Subclasses must implement area()")
    
    def perimeter(self):
        raise NotImplementedError("Subclasses must implement perimeter()")


class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height
    
    def area(self):
        return self.width * self.height
    
    def perimeter(self):
        return 2 * (self.width + self.height)
    
    def __str__(self):
        return f"Rectangle(width={self.width}, height={self.height})"


class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
    
    def area(self):
        return 3.14159 * self.radius ** 2
    
    def perimeter(self):
        return 2 * 3.14159 * self.radius
    
    def __str__(self):
        return f"Circle(radius={self.radius})"


# Duck Typing Example
class Duck:
    def quack(self):
        return "Quack!"
    
    def fly(self):
        return "Flying"


class Person:
    def quack(self):
        return "I'm quacking like a duck!"
    
    def fly(self):
        return "I'm flapping my arms!"


def make_it_quack(thing):
    """This function works with anything that has a quack() method"""
    print(thing.quack())

# Usage
shapes = [Rectangle(4, 5), Circle(3)]

for shape in shapes:
    print(f"{shape} - Area: {shape.area()}, Perimeter: {shape.perimeter()}")

# Duck typing in action
duck = Duck()
person = Person()

make_it_quack(duck)    # Quack!
make_it_quack(person)  # I'm quacking like a duck!

Practice Exercise 2: Employee Hierarchy

Create an employee management system with:

  • Employee base class with name, employee_id, calculate_salary()
  • FullTimeEmployee with monthly salary
  • PartTimeEmployee with hourly rate and hours worked
  • Manager with salary and bonus percentage
  • Demonstrate polymorphism with a list of different employee types
class Employee:
    def __init__(self, name, employee_id):
        self.name = name
        self.employee_id = employee_id
    
    def calculate_salary(self):
        raise NotImplementedError("Subclass must implement calculate_salary")
    
    def __str__(self):
        return f"{self.name} (ID: {self.employee_id})"


class FullTimeEmployee(Employee):
    def __init__(self, name, employee_id, monthly_salary):
        super().__init__(name, employee_id)
        self.monthly_salary = monthly_salary
    
    def calculate_salary(self):
        return self.monthly_salary


class PartTimeEmployee(Employee):
    def __init__(self, name, employee_id, hourly_rate, hours_worked):
        super().__init__(name, employee_id)
        self.hourly_rate = hourly_rate
        self.hours_worked = hours_worked
    
    def calculate_salary(self):
        return self.hourly_rate * self.hours_worked


class Manager(FullTimeEmployee):
    def __init__(self, name, employee_id, monthly_salary, bonus_percentage):
        super().__init__(name, employee_id, monthly_salary)
        self.bonus_percentage = bonus_percentage
    
    def calculate_salary(self):
        base_salary = super().calculate_salary()
        bonus = base_salary * (self.bonus_percentage / 100)
        return base_salary + bonus


# Demonstration of polymorphism
employees = [
    FullTimeEmployee("Alice Smith", "FT001", 5000),
    PartTimeEmployee("Bob Johnson", "PT001", 20, 80),
    Manager("Carol Davis", "MG001", 7000, 15)
]

for emp in employees:
    print(f"{emp} - Salary: ${emp.calculate_salary():.2f}")

1. Class Methods and Static Methods

class Pizza:
    # Class variable - shared by all instances
    total_pizzas = 0
    
    def __init__(self, toppings):
        self.toppings = toppings
        Pizza.total_pizzas += 1
    
    # Instance method - works with instance data
    def describe(self):
        return f"Pizza with {', '.join(self.toppings)}"
    
    # Class method - works with class, not instance
    @classmethod
    def get_total_pizzas(cls):
        return f"Total pizzas created: {cls.total_pizzas}"
    
    @classmethod
    def margherita(cls):
        # Factory method - creates specific type of pizza
        return cls(["tomato", "mozzarella", "basil"])
    
    # Static method - doesn't depend on class or instance
    @staticmethod
    def pizza_fact():
        return "Pizza was invented in Naples, Italy in the 18th century."

# Usage
pizza1 = Pizza(["pepperoni", "mushrooms"])
pizza2 = Pizza.margherita()  # Using class method as factory

print(pizza1.describe())           # Pizza with pepperoni, mushrooms
print(Pizza.get_total_pizzas())    # Total pizzas created: 2
print(Pizza.pizza_fact())          # Pizza was invented in Naples...

2. Magic Methods (Dunder Methods)

class Vector:
    """Demonstrating magic methods"""
    
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    # String representation
    def __str__(self):
        return f"Vector({self.x}, {self.y})"
    
    def __repr__(self):
        return f"Vector({self.x}, {self.y})"
    
    # Arithmetic operations
    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)
    
    def __sub__(self, other):
        return Vector(self.x - other.x, self.y - other.y)
    
    def __mul__(self, scalar):
        return Vector(self.x * scalar, self.y * scalar)
    
    # Comparison operators
    def __eq__(self, other):
        return self.x == other.x and self.y == other.y
    
    def __lt__(self, other):
        return (self.x**2 + self.y**2) < (other.x**2 + other.y**2)
    
    # Length/magnitude
    def __len__(self):
        return 2  # Always 2 for 2D vector
    
    def magnitude(self):
        return (self.x**2 + self.y**2) ** 0.5

# Usage
v1 = Vector(3, 4)
v2 = Vector(1, 2)

print(v1)                   # Vector(3, 4)
print(v1 + v2)              # Vector(4, 6)
print(v1 * 2)               # Vector(6, 8)
print(v1 == v2)             # False
print(v1.magnitude())       # 5.0

3. Abstract Base Classes (ABCs)

from abc import ABC, abstractmethod

class Shape(ABC):
    """Abstract base class for shapes"""
    
    @abstractmethod
    def area(self):
        pass
    
    @abstractmethod
    def perimeter(self):
        pass
    
    def describe(self):
        return f"This shape has area {self.area()} and perimeter {self.perimeter()}"


class Square(Shape):
    def __init__(self, side):
        self.side = side
    
    def area(self):
        return self.side ** 2
    
    def perimeter(self):
        return 4 * self.side


class Triangle(Shape):
    def __init__(self, base, height, side1, side2, side3):
        self.base = base
        self.height = height
        self.side1 = side1
        self.side2 = side2
        self.side3 = side3
    
    def area(self):
        return 0.5 * self.base * self.height
    
    def perimeter(self):
        return self.side1 + self.side2 + self.side3

# Can't instantiate abstract class
# shape = Shape()  # TypeError

square = Square(5)
triangle = Triangle(4, 3, 3, 4, 5)

print(square.area())        # 25
print(triangle.describe())  # This shape has area 6.0 and perimeter 12

â­ Python OOP Best Practices

Essential Best Practices

  • Follow PEP 8: Use CamelCase for class names, snake_case for methods/variables
  • Keep classes focused: Single Responsibility Principle - one class, one job
  • Use properties: Instead of getter/setter methods for simple attributes
  • Document your code: Use docstrings for classes and methods
  • Favor composition over inheritance: "Has-a" is often better than "is-a"
  • Use type hints: Improve code readability and IDE support
  • Implement __repr__: Always provide a useful string representation
  • Keep methods small: Methods should do one thing well
from typing import List, Optional
from datetime import datetime


class Customer:
    """
    Represents a customer in the e-commerce system.
    
    Attributes:
        customer_id: Unique identifier for the customer
        name: Customer's full name
        email: Customer's email address
        join_date: Date when customer joined
        loyalty_points: Points earned through purchases
    """
    
    def __init__(
        self,
        customer_id: str,
        name: str,
        email: str,
        join_date: Optional[datetime] = None
    ):
        self.customer_id = customer_id
        self.name = name
        self.email = email
        self.join_date = join_date or datetime.now()
        self._loyalty_points = 0  # Private attribute
        self._orders: List['Order'] = []  # Type hint with forward reference
    
    @property
    def loyalty_points(self) -> int:
        """Get customer's loyalty points."""
        return self._loyalty_points
    
    @loyalty_points.setter
    def loyalty_points(self, points: int):
        """Set loyalty points with validation."""
        if points < 0:
            raise ValueError("Loyalty points cannot be negative")
        self._loyalty_points = points
    
    def add_order(self, order: 'Order') -> None:
        """Add an order to customer's history."""
        self._orders.append(order)
    
    def get_total_spent(self) -> float:
        """Calculate total amount spent by customer."""
        return sum(order.total_amount for order in self._orders)
    
    def __repr__(self) -> str:
        return (
            f"Customer(id='{self.customer_id}', "
            f"name='{self.name}', "
            f"email='{self.email}')"
        )
    
    def __str__(self) -> str:
        return f"{self.name} <{self.email}>"


# Example of composition over inheritance
class ShoppingCart:
    def __init__(self):
        self.items = []
    
    def add_item(self, product, quantity):
        self.items.append({'product': product, 'quantity': quantity})
    
    def calculate_total(self):
        return sum(item['product'].price * item['quantity'] for item in self.items)


class PremiumCustomer:
    """Uses composition to enhance Customer with premium features"""
    
    def __init__(self, customer: Customer):
        self.customer = customer  # Composition: has-a Customer
        self.discount_rate = 0.1   # 10% discount
        self.free_shipping = True
    
    def apply_discount(self, amount: float) -> float:
        return amount * (1 - self.discount_rate)
