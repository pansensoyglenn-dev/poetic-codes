Python Web Development

Frequently Asked Questions & Architecture Patterns

Django is a "batteries-included" framework, providing an ORM, auth, and admin panel out of the box. Flask is a micro-framework, giving you the bare essentials and letting you choose your own tools.
Use FastAPI if you need high performance, native async/await support, and automatic OpenAPI (Swagger) documentation. It's ideal for modern REST or GraphQL APIs.
WSGI (Web Server Gateway Interface) is for synchronous Python apps. ASGI (Asynchronous Server Gateway Interface) is the successor, supporting WebSockets and asynchronous requests.
Django uses its built-in makemigrations system. For Flask or FastAPI, Alembic (used with SQLAlchemy) is the industry standard for tracking schema changes.
An Object-Relational Mapper allows you to interact with a database using Python classes instead of writing raw SQL. Examples include SQLAlchemy and Django ORM.
Never hardcode secrets! Use a .env file and libraries like python-dotenv or pydantic-settings to load configurations securely.
Jinja2 is a popular templating engine for Python. It allows you to inject Python-like logic (loops, conditionals) directly into your HTML files.
Python's built-in servers (like app.run()) aren't production-ready. Gunicorn (WSGI) and Uvicorn (ASGI) are robust servers designed to handle multiple concurrent workers.
It isolates project dependencies. Tools like venv, Pipenv, or Poetry ensure that Project A doesn't break because of an update in Project B.
Common methods include Session-based auth (common in Django) and JWT (JSON Web Tokens) (preferred for stateless APIs and SPAs).
They handle Background Tasks. If a user uploads a large file or triggers an email, you use Celery to process it "off-thread" so the website stays responsive.
