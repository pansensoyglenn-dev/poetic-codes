Data Structure Schematics

Core and advanced Python containers for efficient data management.

A Tuple is an ordered, immutable collection. Unlike lists, once a tuple is created, you cannot modify its elements. They are defined using parentheses: (1, 2, 3).
Sets are unordered collections of unique elements. They are highly efficient for membership testing and removing duplicates from a sequence. Use curly braces: {1, 2, 3}.
A Dictionary (dict) stores data in key-value pairs. Keys must be hashable (immutable) and unique. Since Python 3.7, dictionaries maintain the insertion order of items.
Located in the collections module, namedtuple assigns names to each position in a tuple, allowing you to access elements by field name as well as index.
A list is optimized for fast fixed-length operations. A deque (double-ended queue) is optimized for fast appends and pops from both ends with O(1) complexity.
collections.Counter is a dictionary subclass for counting hashable objects. It’s perfect for finding the frequency of elements in a list or characters in a string.
A bytearray is a mutable sequence of integers in the range 0 ≤ x < 256. It is essentially a mutable version of a bytes object, useful for binary data manipulation.
A defaultdict never raises a KeyError. It provides a default value for a key that does not exist based on a factory function (like int or list).
The array.array type is more memory-efficient than a list when storing large amounts of basic values (like integers or floats) because it stores them as packed C-style values.
While standard dicts now preserve order, OrderedDict includes specific methods like move_to_end() and is more robust for reordering operations.
The heapq module implements a min-heap on top of a standard list. It ensures the smallest element is always at heap[0], useful for priority queues.
collections.ChainMap groups multiple dictionaries into a single view. When looking up a key, it searches each dictionary in the chain until it finds a match.
