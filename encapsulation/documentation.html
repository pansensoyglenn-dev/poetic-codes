Encapsulation Implementation

In our projects, we utilize "Private" attributes. In Python, this is signified by a single underscore (_attribute).

Note: While Python doesn't strictly enforce private variables like Java or C++, the convention is critical for signaling to other developers that these attributes should not be modified directly.
class Account:
    def __init__(self):
        self._balance = 0 # Encapsulated

ATM System Workflow

The ATM system follows a strict authentication-first logic flow:

  1. Initialization with a 16-digit card number validation.
  2. State check: _is_authenticated must be True for all mutation methods.
  3. Amount validation: Ensuring amount > 0 and amount <= _balance.
