Vegetable Store Inventory System

A clean, educational Python OOP project that demonstrates encapsulation, composition, basic error handling, and clean separation of concerns through a realistic vegetable store point-of-sale simulation.

Project Purpose

This lab teaches core object-oriented programming principles by building a small but meaningful inventory + transaction system — the kind of logic found in real-world retail, warehouse, and e-commerce applications.

Core Learning Objectives

Encapsulation

Protecting object state (stock, price) and exposing behavior only through methods (sell(), get_info()).

Composition

VegetableStore “has-a” collection of Vegetable objects managed via dictionary.

Error Handling

Graceful validation of stock levels and item existence using exceptions.

Single Responsibility

Clear separation: Vegetable manages one item, VegetableStore manages the collection, main() handles UI loop.

Key Features Implemented

  • 10 hardcoded vegetables with realistic Philippine prices (₱)
  • Interactive console menu + transaction loop
  • Stock deduction on successful sale
  • Clear formatting of inventory status
  • Basic user input validation & friendly error messages
