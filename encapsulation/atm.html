Logical Introduction: Why OOP?

Before writing a single line of code, we must understand the Problem-Solution landscape. A banking system requires three pillars: Security, Data Integrity, and Modular Logic.

Security

Protect sensitive data from unauthorized access and modification

Data Integrity

Ensure consistent state and prevent data corruption

Modular Logic

Separate concerns for maintainable, testable code

By using Object-Oriented Programming (OOP), we treat the ATM as a distinct entity. This bundles the user's balance and authentication status directly with the actions they can perform (Deposit, Withdraw), preventing accidental modification by external functions and creating a clear boundary between interface and implementation.

Full ATM Implementation (Python)

This script uses private attributes (_variable) to maintain data integrity and prevent unauthorized access. Each method has a single responsibility, following the SOLID principles.

atm_system.py

#!/usr/bin/env python3
"""
Secure ATM System Implementation
Author: Glenn Pansensoy
Description: A Python OOP demonstration of encapsulation principles
             with proper error handling and security measures.
"""

import sys
from typing import Optional


class ATM:
    """
    Secure ATM class implementing encapsulation principles.
    
    Attributes:
        _card_number (str): Validated 16-digit card number
        _balance (float): Account balance (private)
        _is_authenticated (bool): Authentication state
        _attempts (int): Failed PIN attempts counter
        _max_attempts (int): Maximum allowed attempts
        _correct_pin (str): Predefined correct PIN
    """
    
    def __init__(self, card_number: str, initial_balance: float = 0.0):
        """
        Initialize ATM instance with validation.
        
        Args:
            card_number (str): 16-digit card number
            initial_balance (float): Starting balance
        
        Raises:
            ValueError: If card number validation fails
        """
        self._card_number = self._validate_card_number(card_number)
        self._balance = float(initial_balance)
        self._is_authenticated = False
        self._attempts = 0
        self._max_attempts = 3
        self._correct_pin = "2468"  # In production, use secure storage
        
        print(f"âœ“ ATM initialized for card: {self._mask_card_number()}")

    def _validate_card_number(self, card_number: str) -> str:
        """
        Validate and clean card number input.
        
        Args:
            card_number (str): Raw card number input
        
        Returns:
            str: Cleaned 16-digit card number
        
        Raises:
            ValueError: If card number is invalid
        """
        cleaned = ''.join(filter(str.isdigit, str(card_number)))
        if len(cleaned) != 16:
            raise ValueError("Card number must contain exactly 16 digits.")
        return cleaned

    def _mask_card_number(self) -> str:
        """Return masked card number for security."""
        return f"**** **** **** {self._card_number[-4:]}"

    def authenticate(self) -> bool:
        """
        Handle user authentication with PIN verification.
        
        Returns:
            bool: True if authentication successful
        """
        if self._is_authenticated:
            return True
            
        print(f"\n{'='*40}")
        print(f"Authentication Required")
        print(f"{'='*40}")
        
        while self._attempts < self._max_attempts:
            attempts_left = self._max_attempts - self._attempts
            pin = input(f"Enter PIN ({attempts_left} attempt(s) left): ").strip()
            
            if pin == self._correct_pin:
                self._is_authenticated = True
                print("âœ“ Authentication successful. Access granted.")
                return True
                
            self._attempts += 1
            print("âœ— Invalid PIN. Please try again.")
            
            if self._attempts == self._max_attempts:
                print("\n" + "!"*40)
                print("SECURITY ALERT: Card locked due to multiple failed attempts.")
                print("!"*40)
                return False
        
        return False

    def _get_amount(self, prompt: str) -> Optional[float]:
        """
        Safely get amount input from user with validation.
        
        Args:
            prompt (str): Input prompt message
        
        Returns:
            Optional[float]: Validated amount or None if cancelled
        """
        while True:
            try:
                val = input(f"\n{prompt} (or 'c' to cancel): ").strip().lower()
                
                if val == 'c':
                    print("Transaction cancelled.")
                    return None
                    
                # Remove commas and validate
                amount = float(val.replace(',', ''))
                
                if amount <= 0:
                    print("âœ— Amount must be positive.")
                    continue
                    
                if len(str(amount).split('.')[-1]) > 2:
                    print("âœ— Amount can have at most 2 decimal places.")
                    continue
                    
                return amount
                
            except ValueError:
                print("âœ— Invalid input. Please enter a valid number.")

    def check_balance(self) -> None:
        """Display current account balance if authenticated."""
        if not self._is_authenticated:
            print("âœ— Authentication required.")
            return
            
        print(f"\n{'='*40}")
        print(f"ACCOUNT BALANCE")
        print(f"{'='*40}")
        print(f"Card: {self._mask_card_number()}")
        print(f"Balance: â‚±{self._balance:,.2f}")
        print(f"{'='*40}")

    def deposit(self) -> None:
        """Handle deposit transaction with validation."""
        if not self._is_authenticated:
            print("âœ— Authentication required.")
            return
            
        amount = self._get_amount("Enter deposit amount")
        
        if amount:
            self._balance += amount
            print(f"\nâœ“ Deposit successful!")
            print(f"  Amount: â‚±{amount:,.2f}")
            print(f"  New Balance: â‚±{self._balance:,.2f}")

    def withdraw(self) -> None:
        """Handle withdrawal transaction with balance validation."""
        if not self._is_authenticated:
            print("âœ— Authentication required.")
            return
            
        amount = self._get_amount("Enter withdrawal amount")
        
        if amount:
            if amount <= self._balance:
                self._balance -= amount
                print(f"\nâœ“ Withdrawal successful!")
                print(f"  Amount: â‚±{amount:,.2f}")
                print(f"  New Balance: â‚±{self._balance:,.2f}")
            else:
                print(f"\nâœ— Transaction failed: Insufficient funds.")
                print(f"  Requested: â‚±{amount:,.2f}")
                print(f"  Available: â‚±{self._balance:,.2f}")

    def display_menu(self) -> None:
        """Display main ATM menu."""
        print(f"\n{'='*40}")
        print(f"ATM MAIN MENU")
        print(f"{'='*40}")
        print(f"1. Check Balance")
        print(f"2. Deposit Funds")
        print(f"3. Withdraw Funds")
        print(f"4. Exit")
        print(f"{'='*40}")


def main() -> None:
    """Main program entry point."""
    print(f"\n{'#'*50}")
    print(f"PYTHON ATM SYSTEM v1.0")
    print(f"Implementing Secure Encapsulation Principles")
    print(f"{'#'*50}")
    
    try:
        # Initialize ATM with sample account
        my_atm = ATM("1234567890123456", 5000.0)
        
        # Authentication required before any operations
        if not my_atm.authenticate():
            sys.exit("Program terminated due to security violation.")
        
        # Main transaction loop
        while True:
            my_atm.display_menu()
            choice = input("\nSelect option (1-4): ").strip()
            
            if choice == "1":
                my_atm.check_balance()
            elif choice == "2":
                my_atm.deposit()
            elif choice == "3":
                my_atm.withdraw()
            elif choice == "4":
                print(f"\n{'='*40}")
                print(f"Thank you for using Python ATM System!")
                print(f"Final Balance: â‚±{my_atm._balance:,.2f}")
                print(f"{'='*40}")
                break
            else:
                print("âœ— Invalid selection. Please choose 1-4.")
                
            # Pause for user to read output
            input("\nPress Enter to continue...")
            
    except ValueError as e:
        print(f"\nâœ— Initialization error: {e}")
        sys.exit(1)
    except KeyboardInterrupt:
        print("\n\nâœ— Program interrupted by user.")
        sys.exit(0)
    except Exception as e:
        print(f"\nâœ— Unexpected error: {e}")
        sys.exit(1)


if __name__ == "__main__":
    main()

Logical Analysis & Step-by-Step Explanation

1. Class Initialization & Encapsulation

Significance It establishes a private state. The _balance is shielded from external tampering by using leading underscores, signaling internal use only.

Step-by-Step When ATM() is called, __init__ runs. It cleans the input, validates the 16-digit card requirement, and sets the initial session states.

Key Principle: The underscore prefix (_variable) is a Python convention indicating "private" attributes, though they're still technically accessible (Python trusts developers).

2. State Mutation

Significance Methods are the only authorized way to change the balance, enforcing business rules like "Insufficient Funds" automatically.

Step-by-Step The withdraw() method first verifies the _is_authenticated flag, then calculates if the amount <= _balance before subtracting, ensuring the account never goes negative.

Key Principle: All state changes happen through class methods, never directly. This maintains data integrity and allows for validation at every step.

3. Design Pattern: The Facade

This ATM implementation follows the Facade Pattern - providing a simple interface (deposit, withdraw, check balance) that hides the complexity of validation, authentication, and state management.

Abstraction

Complex operations simplified through method calls

Encapsulation

Data protection through private attributes

Modularity

Independent, reusable components

5 Common Pitfalls & Professional Fixes

Understanding these common mistakes will help you write more robust, production-ready code.

  • Direct State Manipulation: Modifying _balance outside the class bypasses validation logic. Fix: Always use class methods to ensure validation logic runs every time. Implement properties with setters for controlled access.
  • Missing Guard Clauses: Forgetting to return False on failure can lead to security vulnerabilities. Fix: Implement explicit exits if security checks fail. Use early returns to prevent fall-through execution.
  • Floating Point Errors: Using floats for currency math can cause rounding errors in financial applications. Fix: Use Python's decimal module for high-precision apps or represent currency as integer cents.
  • Lack of Input Sanitization: Crashing on non-numeric input creates poor user experience. Fix: Use try...except blocks for all user inputs. Implement input validation before processing.
  • "God Object" Pattern: Putting UI and Database logic inside one class violates Single Responsibility Principle. Fix: Separate concerns into different classes (ATM, Account, TransactionHandler, UI).
Pro Tip: Always write unit tests for your ATM class to ensure each method behaves correctly with various inputs (valid, invalid, edge cases).
