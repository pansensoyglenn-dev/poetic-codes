Architectural Overview & Core Principles

This Python script models a simple yet elegant vegetable store inventory system using Object-Oriented Programming (OOP) principles. It demonstrates encapsulation (bundling data and methods within classes to protect internal state), composition (a store object "owns" multiple vegetable objects), and error handling to ensure data integrity.

At its core, the code promotes modular, maintainable design: The Vegetable class handles individual item logic, the VegetableStore class manages the collection, and the main() function orchestrates user interaction. This separation follows the Single Responsibility Principle (SRP).

Complete Implementation

VEGETABLE_STORE.PY
class Vegetable:
    def __init__(self, name, price_per_kilo, initial_stock):
        self.name = name
        self.price_per_kilo = price_per_kilo
        self.stock = initial_stock 

    def sell(self, quantity):
        if quantity > self.stock:
            raise ValueError(f"Insufficient stock for {self.name}. Available: {self.stock}kg")
        self.stock -= quantity
        return quantity * self.price_per_kilo

    def get_info(self):
        return f"{self.name:<12} | Price: ‚Ç±{self.price_per_kilo}/kg | Stock: {self.stock}kg"

class VegetableStore:
    def __init__(self):
        self.inventory = {
            "asparagus": Vegetable("Asparagus", 180, 100),
            "broccoli": Vegetable("Broccoli", 230, 100),
            "cabbage": Vegetable("Cabbage", 140, 100),
            "garlic": Vegetable("Garlic", 190, 100),
            "onion": Vegetable("Onion", 180, 100),
            "tomatoes": Vegetable("Tomatoes", 160, 100),
            "lettuce": Vegetable("Lettuce", 180, 100),
            "cauliflower": Vegetable("Cauliflower", 220, 100),
            "potatoes": Vegetable("Potatoes", 180, 100),
            "carrots": Vegetable("Carrots", 150, 100)
        }

    def display_menu(self):
        print("\n" + "="*45 + "\nPoetic Codes INVENTORY STATUS\n" + "="*45)
        for veg in self.inventory.values():
            print(veg.get_info())

    def process_transaction(self, item_name, quantity):
        name_key = item_name.lower()
        if name_key not in self.inventory:
            raise ValueError(f"'{item_name}' not found in database.")
        return self.inventory[name_key].sell(quantity)

def main():
    store = VegetableStore()
    while True:
        store.display_menu()
        choice = input("\nBuy (or 'exit'): ").strip()
        if choice.lower() == 'exit': break
        try:
            qty = float(input("Quantity in kilos: "))
            print(f"Total: ‚Ç±{store.process_transaction(choice, qty):,.2f}")
        except ValueError as e:
            print(f"Error: {e}")

if __name__ == "__main__": main()

üß† Logic Analysis: Understanding the Classes

1. The Constructor Logic

Step-by-Step Explanation class Vegetable: defines the blueprint. The __init__ method is called automatically during instantiation. It captures the name, price per kilo, and initial stock, assigning them to instance variables using self.

Significance This constructor is the foundation of encapsulation in OOP. By bundling attributes (data) with the object, it ensures each vegetable maintains its own state independently. In real-world systems, this prevents global variables from causing conflicts.

Logical Analysis Logically, this setup assumes positive numeric inputs for price and stock. The use of self binds data to the instance, enabling object-specific mutations without affecting others.

2. The Sell Method Logic

Step-by-Step Explanation Defines a method to handle selling a given quantity. It first checks if the requested quantity exceeds available stock. If true, it raises a ValueError. Otherwise, it subtracts from stock and returns the total cost.

Significance This method exemplifies data integrity through controlled mutation‚Äîstock can only decrease via this gatekeeper, preventing invalid states.

Logical Analysis The logic follows a "guard clause" pattern: Early exit on invalid input via exception. This is efficient (\( O(1) \) time) and enforces a business invariant: "Stock must always reflect reality."

üéØ Best Practices & Design Patterns

1. Input Validation and Error Handling

Defensive Programming Always validate user input before processing. In our implementation, the sell() method checks stock availability before allowing a transaction. This prevents the system from entering an invalid state.

ROBUST ERROR HANDLING
def sell(self, quantity):
    # Guard clause pattern - fail fast
    if quantity <= 0:
        raise ValueError("Quantity must be positive")
    if quantity > self.stock:
        raise ValueError(f"Insufficient stock for {self.name}")
    
    # Only mutate state after validation
    self.stock -= quantity
    return quantity * self.price_per_kilo

2. Encapsulation: Private vs Public

Design Principle While Python doesn't have true private variables, we can use naming conventions to signal intent. Consider using _private (single underscore) for internal use or __very_private (double underscore) for name mangling.

ENHANCED ENCAPSULATION
class Vegetable:
    def __init__(self, name, price_per_kilo, initial_stock):
        self._name = name  # Protected (convention)
        self._price_per_kilo = price_per_kilo
        self.__stock = initial_stock  # Private (name mangled)
    
    @property
    def name(self):
        """Read-only access to name"""
        return self._name
    
    @property
    def stock(self):
        """Read-only access to stock"""
        return self.__stock
    
    @property
    def price(self):
        """Read-only access to price"""
        return self._price_per_kilo

3. Single Responsibility Principle

Design Insight Each class should have one reason to change. The Vegetable class manages individual items, the VegetableStore class manages the collection, and main() handles user interaction. This separation makes testing and maintenance easier.

From Dictionary to Database

Scalability Path While our current implementation uses a Python dictionary for simplicity, production systems typically use databases:

DATABASE INTEGRATION EXAMPLE
import sqlite3
from typing import Optional

class DatabaseVegetableStore:
    def __init__(self, db_path: str = "store.db"):
        self.conn = sqlite3.connect(db_path)
        self._init_database()
    
    def _init_database(self):
        """Create tables if they don't exist"""
        self.conn.execute('''
            CREATE TABLE IF NOT EXISTS vegetables (
                id INTEGER PRIMARY KEY,
                name TEXT UNIQUE NOT NULL,
                price_per_kilo REAL NOT NULL,
                stock REAL NOT NULL
            )
        ''')
        self.conn.commit()
    
    def add_vegetable(self, name: str, price: float, stock: float):
        """Add new vegetable to inventory"""
        self.conn.execute(
            'INSERT INTO vegetables (name, price_per_kilo, stock) VALUES (?, ?, ?)',
            (name, price, stock)
        )
        self.conn.commit()
    
    def update_stock(self, name: str, quantity: float) -> bool:
        """Update stock after transaction"""
        cursor = self.conn.execute(
            'SELECT stock FROM vegetables WHERE name = ?',
            (name,)
        )
        result = cursor.fetchone()
        
        if not result or result[0] < quantity:
            return False
        
        self.conn.execute(
            'UPDATE vegetables SET stock = stock - ? WHERE name = ?',
            (quantity, name)
        )
        self.conn.commit()
        return True

Unit Testing Best Practices

Professional Standards In production environments, every class and method should have corresponding unit tests. Here's how to test our vegetable store:

test_vegetable_store.py
import unittest
from vegetable_store import Vegetable, VegetableStore

class TestVegetable(unittest.TestCase):
    def setUp(self):
        """Create a test vegetable before each test"""
        self.veg = Vegetable("Tomato", 150, 50)
    
    def test_initialization(self):
        """Test if vegetable initializes correctly"""
        self.assertEqual(self.veg.name, "Tomato")
        self.assertEqual(self.veg.price_per_kilo, 150)
        self.assertEqual(self.veg.stock, 50)
    
    def test_successful_sell(self):
        """Test selling within available stock"""
        cost = self.veg.sell(10)
        self.assertEqual(cost, 1500)  # 10 * 150
        self.assertEqual(self.veg.stock, 40)  # 50 - 10
    
    def test_insufficient_stock(self):
        """Test that selling more than available raises error"""
        with self.assertRaises(ValueError):
            self.veg.sell(60)  # More than 50 available
    
    def test_get_info(self):
        """Test info string generation"""
        info = self.veg.get_info()
        self.assertIn("Tomato", info)
        self.assertIn("‚Ç±150", info)

class TestVegetableStore(unittest.TestCase):
    def setUp(self):
        """Create a test store before each test"""
        self.store = VegetableStore()
    
    def test_store_initialization(self):
        """Test if store initializes with correct items"""
        self.assertIn("tomatoes", self.store.inventory)
        self.assertEqual(len(self.store.inventory), 10)
    
    def test_process_transaction(self):
        """Test successful transaction"""
        cost = self.store.process_transaction("broccoli", 5)
        self.assertEqual(cost, 1150)  # 5 * 230
    
    def test_invalid_item(self):
        """Test transaction with non-existent item"""
        with self.assertRaises(ValueError):
            self.store.process_transaction("banana", 5)

if __name__ == '__main__':
    unittest.main()

3. Concurrency Considerations

Thread Safety In multi-threaded environments (like web applications), you need to protect shared state:

THREAD-SAFE IMPLEMENTATION
import threading

class ThreadSafeVegetableStore:
    def __init__(self):
        self.inventory = {
            "broccoli": Vegetable("Broccoli", 230, 100),
            # ... other vegetables
        }
        self._lock = threading.Lock()
    
    def process_transaction(self, item_name, quantity):
        """Thread-safe transaction processing"""
        with self._lock:  # Acquire lock before modifying state
            name_key = item_name.lower()
            if name_key not in self.inventory:
                raise ValueError(f"'{item_name}' not found")
            return self.inventory[name_key].sell(quantity)
