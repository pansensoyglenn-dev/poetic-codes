Advanced Python Design Patterns

Implementation details for scalable, maintainable, and decoupled Python systems. These patterns optimize for Python's dynamic nature while maintaining strict architectural integrity.

Creational Patterns

Abstract Factory

Intent: Provide an interface for creating families of related objects.

Useful when a system must be independent of how its products are created, or when you need to enforce a suite of products that work together.

class UIFactory:
    def create_button(self): pass
    def create_menu(self): pass

class DarkModeFactory(UIFactory):
    def create_button(self): return DarkButton()

Prototype

Intent: Create new objects by copying an existing instance.

In Python, this is efficiently handled via the copy module. Useful when object creation is expensive.

import copy

class Registry:
    def clone(self, obj):
        return copy.deepcopy(obj)
Structural Patterns

Facade Pattern

Intent: Provide a unified interface to a set of interfaces in a subsystem.

Crucial for simplifying complex library interactions into a single high-level API for the client.

Proxy Pattern

Intent: Provide a placeholder for another object to control access.

Commonly used for lazy initialization, logging, or managing access to sensitive resources.

class RealImage:
    def display(self): pass

class ProxyImage:
    def __init__(self):
        self._real = None
    def display(self):
        if not self._real:
            self._real = RealImage()
        self._real.display()
Behavioral Patterns

Command Pattern

Intent: Encapsulate a request as an object.

Allows for parameterization of clients with queues, undoable operations, and request logging.

class Command:
    def execute(self): pass

class SaveCommand(Command):
    def execute(self):
        editor.save()

State Pattern

Intent: Allow an object to alter its behavior when its internal state changes.

Avoids massive if/else or match statements by delegating behavior to state objects.

    def handle(self): pass
